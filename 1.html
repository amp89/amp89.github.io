Switch and If/Else

Switch and if/else statements have very little differences when it comes to the functionality of the code. Below are examples of code for first a switch statement, and then an if/else statement. They give the exact same results.

Switch Exampe: 
switch (x){
  case 0: y = "zero";
    break;
  case 1: y = "one";
    break;
  case 2: y = "two";
    break;
  case 3: y = "three";
    break;
  case 4: y = "four";
    break;
  default: y = "error";
    break;
}

If/Else Example: 
if (x==0)
{
  y = "zero";
}
else if (x == 1)
{
  y = "one";
}
else if (x==2)
{
  y = "two";
}
else if (x==3)
{
  y = "three";
}
else if (x==4)
{
  y = "four";
}
else
{
  y = "error";
}

Why would a "switch" statement be useful when it works the same as an "if/else" statement? It can be difficult to think of reasons to use a switch statement when an if/else can do the same thing and more, however, there are a few instances where a "switch" can make more sense than an "if/else."

One situation in which a "switch" works best is reducing errors when a decision must be made when comparing one variable. As you can see in the code, the "x" is only written one time in the switch code, where as it is written 5 times in the if/else code. It is much quicker and less prone to errors to write switch(x) than to write (x==z) five times.

On the other hand, if more complicated decision making must be made (for example, x == something and z == something else), an if/else statement would be much simpler.

Another reason switch would be used instead of an if else statement is readability. This might be my own personal bias, however, when I compare the two codes that are identical in functionality, it seems to take less time to comprehend what is going on in the switch statement.
